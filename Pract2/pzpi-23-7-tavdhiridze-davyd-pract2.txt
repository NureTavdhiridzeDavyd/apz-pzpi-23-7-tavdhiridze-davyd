3 ОПИС ВИКОНАНОЇ РОБОТИ
     Загальні відомості про Amazon Web Services
     Amazon Web Services (AWS) є основою для побудови сучасних хмарних рішень, надаючи понад 200 сервісів для мільйонів клієнтів у всьому світі. У межах роботи розглянуто ключові моделі обслуговування, такі як IaaS (інфраструктура), PaaS (платформа) та SaaS (програмне забезпечення), що дозволяють гнучко підбирати рівень абстракції для бізнес-завдань. Розуміння цих моделей є критичним для проектування систем, оскільки воно визначає зону відповідальності між провайдером та розробником.
     Порівняння традиційної та хмарної інфраструктур
     Важливим етапом дослідження стало порівняння класичного підходу до серверного обладнання та хмарних технологій. Традиційна інфраструктура пов'язана з високими капітальними витратами та складним масштабуванням, тоді як AWS пропонує модель оплати за фактичне використання та миттєве регулювання ресурсів. Такий підхід мінімізує фінансові ризики та забезпечує високу доступність сервісів незалежно від навантаження.
     Архітектурні принципи хмарної платформи
     Побудова надійних систем в AWS базується на фундаментальних принципах: масштабованості, еластичності, відмовостійкості та розподіленій архітектурі. Масштабованість дозволяє змінювати потужність ресурсів без зупинки роботи сервісу, а еластичність автоматизує цей процес відповідно до попиту. Відмовостійкість досягається шляхом дублювання компонентів, що гарантує стабільність системи навіть у разі технічних збоїв.
     Аналіз основних сервісів AWS
     У роботі проаналізовано базовий набір інструментів, що забезпечують функціонування хмари: обчислювальні потужності (EC2, Lambda), системи збереження даних (S3), реляційні та NoSQL бази даних (RDS, DynamoDB), а також засоби мережевої ізоляції (VPC) та управління доступом (IAM). Кожен сервіс виконує специфічну роль у забезпеченні продуктивності та безпеки інфраструктури.
     Архітектура мікросервісів у хмарному середовищі
     Сучасний підхід до розробки в AWS передбачає використання мікросервісів, які взаємодіють через API Gateway та черги повідомлень (SQS/SNS). Це дозволяє кожному компоненту системи розвиватися та масштабуватися незалежно. Використання контейнеризації через ECS забезпечує зручне розгортання бізнес-логіки без необхідності ручного управління серверами.
     Масштабування та забезпечення відмовостійкості
     Для підтримки стабільної роботи застосунків використовуються механізми Auto Scaling та розгортання в декількох зонах доступності (Multi-AZ). Автоматичне регулювання кількості інстансів базується на метриках CloudWatch, що дозволяє оптимізувати витрати. Синхронна реплікація баз даних у межах різних дата-центрів гарантує збереження даних та мінімальний час відновлення після аварій.
     Проектування типового веб-застосунку
     Практична частина роботи включає аналіз архітектури веб-застосунку, де поєднуються статичний контент у S3, динамічна обробка на EC2 та швидка доставка через CloudFront CDN. Розподіл трафіку через Application Load Balancer та використання IAM-ролей забезпечує високу безпеку та оптимальне навантаження на систему.
     Програмна взаємодія через Boto3
     У межах виконання роботи продемонстровано використання офіційного SDK для Python — бібліотеки boto3. Наведені приклади коду ілюструють програмне завантаження об'єктів у сховище S3 та дистанційний виклик Lambda-функцій з передачею JSON-даних. Це підкреслює можливість повної автоматизації хмарних процесів без використання консолі управління.
     Інфраструктура як код (Terraform)
     Окрему увагу приділено декларативному підходу до управління ресурсами за допомогою Terraform. Опис інфраструктури у вигляді коду дозволяє версіонувати конфігурації в Git, забезпечуючи ідентичність середовищ розробки та продакшну. Приклади створення EC2-інстансів та S3-бакетів через конфігураційні файли демонструють ефективність автоматизації розгортання.
     Висновки щодо ефективності AWS
     Завершальний етап роботи підтверджує, що AWS є лідером ринку завдяки широкому спектру інструментів та надійним архітектурним принципам. Використання мікросервісів, автоматизації через Boto3 та Terraform дозволяє створювати гнучкі та економічно вигідні рішення для сучасного бізнесу.

4 ВИСНОВКИ
     У ході виконання роботи було розглянуто архітектуру та ключові сервіси хмарної платформи Amazon Web Services (AWS). Проаналізовано призначення платформи, її основні компоненти та фундаментальні принципи побудови надійних хмарних систем.
     Було розглянуто структуру глобальної інфраструктури AWS, зокрема ролі обчислювальних сервісів (EC2, Lambda), систем збереження даних (S3), баз даних (RDS, DynamoDB) та засобів мережевої безпеки (VPC, IAM). Визначено, що головною ідеєю використання хмарної платформи є перехід від капітальних до операційних витрат, що дозволяє реалізувати механізми миттєвого масштабування, високої доступності та ефективного управління ресурсами через розподілену архітектуру.
     Також було розглянуто приклади програмної реалізації управління хмарною інфраструктурою за допомогою бібліотеки Boto3 для мови Python та інструменту декларативного опису ресурсів Terraform. Це дозволило продемонструвати практичне застосування концепції «Інфраструктура як код» (IaC) та зрозуміти переваги автоматизації процесів розгортання й версіонування систем.
     У результаті виконання роботи було сформовано розуміння принципів побудови масштабованих мікросервісних систем у хмарі та значення AWS для створення сучасної, гнучкої та відмовостійкої архітектури програмного забезпечення.
